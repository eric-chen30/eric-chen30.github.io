# CSR、SSR、SPA

# CSR

什么是 CSR

页面由 JS 渲染，JS 运行于浏览器端，称为客户端渲染。Vue、React 都是基于这种模式。

优缺点

  • 优点：
  1. 前后端分离的架构，有利于开发效率的提升
  • 缺点：
  1. 首屏渲染时间较长
  2. 不利于 SEO (搜索引擎优化)

基于 React 的 csr 渲染流程

浏览器下载 HTML 文档 ——> 下载 JS——> 运行 react 代码构建页面 ——> 展示页面

# SSR

什么是 SSR

把 Vue 组件渲染为服务器的 HTML 字符串，将他们直接发送到浏览器，最后将静态标记 混和 为客户端上完成全交互应用程序。

优缺点

  • 优点：
  1. SEO 搜索引擎优化
  2. 首屏显示时间
  • 缺点：
  1. 开发逻辑复杂
  2. 开发条件有所限制，比如一些生命周期、第三方库不能使用
  3. 服务器的负载增大
