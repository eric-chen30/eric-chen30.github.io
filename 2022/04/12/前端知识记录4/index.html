# 前端知识点汇总 (四)

# 正则表达式中的符号

  • ？：问号之前的字符只出现零次或一次

  • +：加号之前的一个字符至少出现一次

  • *：星号可以替换为任意字符串

  • ()：将括号内容看作一个整体执行上述匹配模式

  • \w：匹配字母、数字、下划线。等价于 [A-Za-z0-9_]

  • .：匹配除换行符（\n、\r）之外的任何单个字符

  • [A-Z]：[A-Z] 表示一个区间，匹配所有大写字母，[a-z] 表示所有小写字母

  • [ABC]：匹配除后面字符的所有字符

  • [ABC]：匹配括号里的所有字符

# box-shadow 属性

  • offset-x 阴影水平方向偏移，正值为右偏移，负值为左偏移
  • offset-y 阴影垂直方向偏移，正值为下偏移，负值为上偏移
  • blur 阴影模糊半径，值越大，越模糊，更朦胧，0 意味着阴影是固态而锋利的
  • spread 阴影扩展半径，值越大，阴影扩展范围越大
  • color 阴影的颜色
  • position 外 / 内阴影 默认为外，设置值为 inset 时，为内阴影

# px、em、rem 的区别

  • px：绝对单位长度，相对于显示器屏幕的分辨率而言的
  • em：相对长度单位，值不固定，相对于父元素字体大小
  • rem：相对于 html 根元素的 font-size

rem 的缺点：屏幕越小的移动设备如果用了 rem 肯定文字就越小，就会导致文章看的时候特别费眼。

# 解析过程

解析过程分为 【语法检查阶段】 和 【运行阶段】

  • 语法检查阶段 包括 【词法分析】 和 【语法分析】
  • 运行阶段 包括 【预解析】 和 【运行阶段】
  1. 词法分析

词法分析会将 js 代码中的字符串分割为有意义的代码块，这些代码块我们可以称之为 “词法单元”。

  1. 语法分析

语法分析在这个过程中会将词法单元流转换成一颗 抽象语法树 (AST)。

  1. 预解析

在 js 文件或 script 里面的代码在正式开始执行之前，会进行一些解析工作。比如在全局上寻找 var 关键字声明的变量或者通过 function 关键字声明的函数，找到全局变量或者全局函数后，会进行作用域提升，但提升后不会立即进行赋值操作，此时默认值为 undefined

  1. 预编译
  • 全局预编译，未调用的函数不进行编译
  • 只有 var 和 function 关键字声明的变量和函数才会发生作用域提升
  • 在所在的作用域提升后，不会扩展到其他作用域
  • 预编译后会顺序执行代码

# 外边距折叠问题

多个相邻元素之间在垂直方向上的 margin 会发生重叠

  • 两个相邻元素的 margin 都为正值时，折叠结果为两者之间较大的值
  • 两个相邻元素的 margin 都为负值时，折叠结果是两者绝对值中的较大值
  • 两个外边距为一正一负时，折叠结果是两者的和

# 浮动清除

高度塌陷：子元素都设置为浮动且父元素没有设置高度的时候会产生高度塌陷

  • 在浮动元素后添加一个带 clear 属性的空元素 可以清除浮动
  • 使用 CSS 的 overflow 属性，将 overflow 设置为 hidden 或者 auto
  • 给父元素也添加浮动属性，但这样会影响布局，不建议使用
  • 万能去浮动：添加一个伪类

# JavaScript 基本数据类型

  • String
  • Number
  • undefined 变量声明了但没有赋值 转换为布尔值为 false number (undefined) 结果为 NaN
  • null 变量定义了但为空 转换为布尔值为 false number (null) 结果为 0
  • boolean
  • Object
  • Symbol
  • bigInt

NaN：

  • typeof (NaN) 结果为 number 类型
  • NaN 是唯一一个不等于自身的量 NaN ！== NaN
  • 检验是否为 NaN 可以使用 方法 isNaN 进行判断

# 比较两个值是否相等

  • 相等运算符 == ： 会自动转换数据类型，不是严格的相等
  • 严格相等运算 === ： NaN 不等于自身，并且 +0 等于 -0

针对上面两种相等，ES6 在对象上新增了 Object.is() 方法，也就是同值相等算法，判断两个值是否严格相等，解决了严格相等运算符的缺陷

# Object.assign()

# 基本用法

该方法用于对象的合并，将源对象的所有可枚举属性，复制到目标对象

  • 第一个参数为目标对象，后面的参数都是源对象
  • 如果具有同名属性，后面的属性会覆盖掉前面的属性
  • 只有一个参数的时候，参数为对象，返回目标对象本身；如果不是对象，会先转换为对象，然后返回
  • 如果将 null 和 undefined 当作参数传入方法中，由于两者无法转换为对象，因此会报错
  • 限制：只拷贝源对象自身属性（不包括继承属性）、也不包括不可枚举属性

# 注意点

  1. 浅拷贝，不是深拷贝，如果源对象某个属性的值是对象，那么目标对象拷贝得到的是这个对象的引用

  2. 同名属性替换：遇到同名属性，处理方法是替换而不是添加

  3. 可以用来处理数组，但是会把数组视为对象

  4. 取值函数的处理，不会复制取值函数，而是将取值函数的结果进行复制

# ES6 对象新增的一些其他方法

  • Object.keys () 返回一个数组，包含对象自身可枚举的属性的键名
  • Object.values () 返回一个数组，包含对象自身可枚举的属性的值
  • Object.entries () 返回一个数组，包含对象自身可枚举的键值对
  • Object.fromEntries () 将一个键值对数组转换成一个对象

# 数组实例方法：find () 和 findIndex ()

# find()

找出第一个符合条件的数组成员。参数是一个回调函数，所有数组成员一次执行回调，回调包含三个参数，（当前值，当前索引，原数组）

  • 找到第一个返回值为 true 的成员并返回
  • 如果没有符合条件的数组成员，返回 undefined

# findIndex()

返回第一个符合条件的数组成员的位置。参数是一个回调函数，所有数组成员一次执行回调，回调包含三个参数，（当前值，当前索引，原数组）

  • 找到第一个符合条件的成员，返回成员索引
  • 如果没有符合条件的数组成员，返回 -1

对于 find () 和 findIndex () 可以接收第二个参数，用来绑定回调函数中的 this 对象

这两个方法弥补了数组 indexOf 方法的不足，无法发现 NaN
